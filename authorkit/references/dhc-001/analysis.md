## 6. KESE KIT 연계 분석

### 기존 KESE KIT과의 겹침/보완 관계

| 디지털헬스케어 영역 | KESE KIT 기존 장 | 관계 |
|---|---|---|
| 서버 보안 (Part1 제5장 3절) | 3~4장 (Unix/Windows 서버) | 겹침 — 헬스케어 특수성 추가 필요 |
| WEB 서버 보안 | 5장 (웹 서비스) | 겹침 — 의료 웹 서비스 맥락 추가 |
| DBMS 보안 | 7장 (데이터베이스) | 겹침 — 의료정보 DB 보안 강화 |
| 시큐어코딩 | 28~33장 (시큐어코딩) | 겹침 — 헬스케어 코드 보안 예시 추가 |
| AI 데이터 학습 보안 | 24~27장 (AI 보안) | 겹침 — 의료 AI 특수 요구사항 |
| **기기 보안요구사항** | **없음** | **신규 — IoT/의료기기 보안** |
| **서비스 유형별 보안** | **없음** | **신규 — 10개 헬스케어 서비스 보안** |
| **물리적 보호** | 19장 (물리보안) | 보완 — 의료기기 물리적 보안 |

### 신규 콘텐츠 (KESE KIT에 없는 내용)

1. **의료기기 인증·인가** — 기기 고유 ID, MAC/IP 제한, 응급 접근, BLE/Wi-Fi 보안모드
2. **디지털헬스케어 서비스 10개 유형** — 각 유형별 고유 보안위협·요구사항
3. **의료 데이터 비식별화** — PHI/PHR 특수 보호, 재식별화 방지
4. **디지털치료제(DTx) 보안** — 소프트웨어 기반 의료기기 보안
5. **원격수술 로봇 보안** — 비대면 시술 시 실시간 보안 요구사항
6. **의료 AI 데이터 학습** — 편향 데이터 필터링, AI 가이드라인 준수

## 7. 이슈 및 참고사항

1. Part1(2020.12)과 Part2(2021.12)는 1년 차이로 보완 관계
2. Part1은 기기+인프라 중심, Part2는 서비스 유형 중심 — 상호보완적
3. Part2의 보안요구사항 45개는 Part1의 63개와 일부 중복 (인증, 암호 등)
4. 식품의약품안전처 의료기기 사이버보안 허가·심사와 연계 가능
5. 개인정보보호법 제17조(제3자 제공) 등 법적 준거성 내용 포함
